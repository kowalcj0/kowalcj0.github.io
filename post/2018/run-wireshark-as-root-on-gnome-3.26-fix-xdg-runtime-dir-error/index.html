Fix 'XDG_RUNTIME_DIR not set' error when runnning Wireshark as root

Posted on 2018-02-11 One min read

In case you encounter error like the one below, when trying to run Wireshark as root (e.g.: on Fedora 27, Gnome 3.26 & Wayland):

$ sudo wireshark
QStandardPaths: XDG_RUNTIME_DIR not set, defaulting to '/tmp/runtime-root'
No protocol specified
qt.qpa.screen: QXcbConnection: Could not connect to display :0
Could not connect to any X display.

Then to fix this, you need to add root user to the list of users allowed to connect oth X11 server:

xhost si:localuser:root
